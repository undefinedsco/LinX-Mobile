# AGENTS.md

This file applies to the whole repository.

## Role

Act as a senior mobile engineer. Treat this repository as a multi-surface mobile
codebase with a React Native app at the root and a standalone native SwiftUI app
under `apple/`.

When working on iOS, Swift, SwiftUI, UIKit, networking, authentication, storage,
or runtime integration, use production-grade iOS engineering practices and keep
changes aligned with the existing module boundaries.

## Repository Surfaces

```text
.
|-- App.tsx
|-- index.js
|-- package.json
|-- src/
|-- ios/
|-- android/
`-- apple/
```

### React Native Surface

The root app is `LinXMobile`.

Use this surface for cross-platform React Native work:

- `App.tsx`
- `index.js`
- `src/`
- `ios/`
- `android/`
- `__tests__/`

Current state:

- React Native `0.85.3`
- React `19.2.3`
- TypeScript
- Hermes enabled
- React Native New Architecture enabled
- iOS host app generated by React Native CLI
- Android host app generated by React Native CLI

### Native Apple Surface

The `apple/` app is `LinXApple`.

Use this surface for native SwiftUI work:

- `apple/LinXApple/Sources/AppCore`
- `apple/LinXApple/Sources/Auth`
- `apple/LinXApple/Sources/ChatUI`
- `apple/LinXApple/Sources/PodData`
- `apple/LinXApple/Sources/Runtime`

Follow `apple/AGENTS.md` for all work inside `apple/`.

## Build And Run

Install JavaScript dependencies:

```sh
npm install
```

Start Metro:

```sh
npm start
```

Run React Native iOS:

```sh
npm run ios
```

Run React Native Android:

```sh
npm run android
```

Install React Native iOS Pods:

```sh
bundle install
cd ios
bundle exec pod install
```

Open the native Apple app:

```sh
open apple/LinXApple.xcodeproj
```

## Tests

Run React Native tests:

```sh
npm test
```

Run native Apple tests:

```sh
xcodebuild test \
  -project apple/LinXApple.xcodeproj \
  -scheme LinXApple \
  -destination 'platform=iOS Simulator,name=iPhone 16'
```

## Change Boundaries

- Do not mix React Native and native Apple changes unless the requested feature
  requires cross-surface integration.
- Keep React Native feature code under `src/`.
- Keep React Native iOS host changes under `ios/`.
- Keep React Native Android host changes under `android/`.
- Keep native SwiftUI app changes under `apple/LinXApple/Sources`.
- Do not manually edit generated Xcode project files when the change belongs in
  an XcodeGen `project.yml` file.
- Do not commit secrets, tokens, refresh tokens, WebIDs, local paths, or private
  credentials.

## Documentation Rules

- Keep the root `README.md` focused on the full repository.
- Keep `apple/README.md` focused on the standalone native Apple app.
- Keep the root `AGENTS.md` focused on repository-wide rules.
- Keep `apple/AGENTS.md` focused on native iOS implementation rules.

## Engineering Rules

- Prefer existing local architecture over new abstractions.
- Keep changes scoped to the requested behavior.
- Add tests when changing parsing, auth, storage path derivation, SPARQL
  generation, model selection, or non-trivial UI orchestration.
- Preserve Swift 6 strict concurrency compatibility in the native Apple app.
- Preserve React Native New Architecture compatibility in the root app.
