# LinX Mobile

LinX Mobile is a multi-surface mobile repository. The repository currently has
two distinct app surfaces:

- `LinXMobile`: a React Native app at the repository root with iOS and Android
  native shells.
- `LinXApple`: a standalone native SwiftUI iOS app under `apple/` with the main
  product logic implemented today.

The React Native surface is still close to the default template. The native
Apple surface contains the complete authentication, Pod-backed chat history, and
LinX runtime chat integration.

## Repository Layout

```text
.
|-- App.tsx                         # React Native root component
|-- index.js                        # React Native AppRegistry entry
|-- package.json                    # React Native scripts and JS dependencies
|-- src/                            # Reserved React Native feature modules
|-- ios/                            # React Native iOS host app
|-- android/                        # React Native Android host app
|-- __tests__/                      # React Native Jest tests
`-- apple/                          # Standalone native SwiftUI iOS app
    |-- project.yml                 # XcodeGen project source
    |-- LinXApple.xcodeproj         # Generated Xcode project
    |-- LinXApple/
    |   |-- Resources/
    |   `-- Sources/
    |       |-- AppCore/
    |       |-- Auth/
    |       |-- ChatUI/
    |       |-- PodData/
    |       `-- Runtime/
    |-- LinXAppleTests/
    `-- LinXAppleUITests/
```

## Key Technologies

### React Native Surface

- React Native `0.85.3`
- React `19.2.3`
- TypeScript
- Metro
- Jest
- Hermes enabled
- React Native New Architecture enabled
- CocoaPods for the React Native iOS host
- Kotlin and Gradle for the Android host

### Native Apple Surface

- SwiftUI
- Swift 6
- Strict Swift Concurrency set to `complete`
- iOS 17 deployment target
- XcodeGen
- Swift Package Manager
- AppAuth for OIDC authorization code + PKCE
- Keychain Services for secure session persistence
- URLSession with async/await
- Exyte Chat for chat UI
- SwiftOpenAI for OpenAI-compatible runtime calls
- MarkdownView for assistant markdown rendering
- RDF Turtle resources and SPARQL for Pod-backed chat storage

## React Native App

The React Native app starts from:

- `index.js`
- `App.tsx`

Current behavior:

- Registers the `LinXMobile` component.
- Renders the default `@react-native/new-app-screen`.
- Wraps the app with `SafeAreaProvider`.
- Uses native iOS and Android shells generated by the React Native CLI.

The `src/` directory is present for future feature modules, but there is no
substantial React Native business logic in it yet.

## Native iOS App

The native iOS app under `apple/` is organized by feature and infrastructure
boundary.

```text
apple/LinXApple/Sources
|-- AppCore
|-- Auth
|-- ChatUI
|-- PodData
`-- Runtime
```

### AppCore

Owns app startup, route selection, shared domain models, constants, and small
platform helpers.

Important files:

- `LinXAppleApp.swift`: SwiftUI `@main` app entry.
- `RootView.swift`: switches between launching, login, and chat states.
- `DomainModels.swift`: launch phase, message models, thread models, errors.
- `AppConstants.swift`: issuer URL, runtime URL, redirect URL, default model,
  default chat ID, and agent identity.

### Auth

Owns OIDC authentication and secure session storage.

Flow:

```text
OIDC discovery
--> dynamic client registration
--> authorization code + PKCE via AppAuth
--> ID token WebID extraction
--> OIDAuthState + metadata persisted in Keychain
--> authenticated app phase
```

Important files:

- `AuthSessionController.swift`: main auth state machine.
- `OIDCDiscoveryClient.swift`: loads OIDC discovery metadata.
- `DynamicClientRegistrar.swift`: registers a public native client.
- `PKCECoordinator.swift`: builds the authorization request.
- `KeychainSessionStore.swift`: stores auth state and session metadata.
- `JWTUtilities.swift`: extracts WebID from `webid` or URL-shaped `sub`.

### ChatUI

Owns SwiftUI chat screens and the chat experience state model.

Important files:

- `ChatExperienceModel.swift`: coordinates bootstrap, thread loading, message
  sending, streaming updates, cancellation, retry, and repository writes.
- `ChatScene.swift`: hosts Exyte `ChatView`.
- `ThreadListView.swift`: shows existing chat threads.
- `LoginView.swift`: login screen.
- `AssistantMarkdownBubble.swift`: renders assistant markdown responses.
- `ExyteMessageAdapter.swift`: maps domain messages into Exyte messages.

### PodData

Owns the Pod-backed storage model.

Storage shape:

```text
{podBase}/.data/
|-- chat/
|   `-- cli-default/
|       |-- index.ttl
|       `-- yyyy/mm/dd/messages.ttl
`-- agents/
    `-- linx-cli-assistant.ttl
```

Important files:

- `PodBootstrapper.swift`: creates required containers and seed resources.
- `PodStoragePaths.swift`: resolves Pod URLs and resource paths from WebID.
- `PodSPARQLClient.swift`: sends authorized HEAD, PUT, PATCH, and SPARQL query
  requests.
- `PodSPARQLBuilder.swift`: builds Turtle and SPARQL update/query strings.
- `PodChatRepository.swift`: repository API for chat threads and messages.

### Runtime

Owns model catalog lookup and chat completion calls.

Important files:

- `LinxModelCatalogClient.swift`: fetches `/v1/models` and prefers
  `linx-lite` when available.
- `LinxOpenAIChatService.swift`: streams chat completions through the LinX
  OpenAI-compatible runtime and falls back to a non-streaming completion when
  streaming fails.

## Main Native Flow

```text
LinXAppleApp
--> AuthSessionController.restore()
--> ChatExperienceModel.bootstrapIfNeeded()
--> PodBootstrapper.bootstrap()
--> ChatScene
--> ChatExperienceModel.enqueueSend()
--> PodChatRepository append user message
--> PodChatRepository append assistant placeholder
--> LinxOpenAIChatService stream reply
--> ChatExperienceModel updates UI and patches Pod
```

## Setup

Install JavaScript dependencies:

```sh
npm install
```

Install iOS CocoaPods for the React Native host:

```sh
bundle install
cd ios
bundle exec pod install
```

Generate the native Apple Xcode project after editing `apple/project.yml`:

```sh
cd apple
xcodegen generate
```

## Run

Start Metro:

```sh
npm start
```

Run React Native iOS:

```sh
npm run ios
```

Run React Native Android:

```sh
npm run android
```

Run the native Apple app:

```sh
open apple/LinXApple.xcodeproj
```

Then select the `LinXApple` scheme in Xcode and run on an iOS simulator or
device.

## Test

Run React Native tests:

```sh
npm test
```

Run native Apple unit tests from Xcode using the `LinXApple` scheme, or from the
command line:

```sh
xcodebuild test \
  -project apple/LinXApple.xcodeproj \
  -scheme LinXApple \
  -destination 'platform=iOS Simulator,name=iPhone 16'
```

## Development Notes

- Treat the React Native app and `apple/LinXApple` as separate app surfaces.
- Add cross-platform features under `src/` only when the feature is intended for
  the React Native app.
- Add native SwiftUI features under `apple/LinXApple/Sources`.
- Keep authentication code isolated in `Auth`.
- Keep Pod storage and SPARQL details isolated in `PodData`.
- Keep OpenAI-compatible runtime calls isolated in `Runtime`.
- Keep screen state and user interaction orchestration in `ChatUI`.
- Do not hardcode tokens or user-specific WebIDs.
- Preserve Swift 6 strict concurrency compatibility in the native Apple app.
